# 黃刀鎮極光旅 2027：下一步執行計畫

## 目前判斷

M2 靜態決策基線已通過，專案現在不需要重做架構，也不應改動 Repository、Gate 或核心任務順序。下一階段的重點是把「可比較、可重查、可決策」補齊，並避免把尚未公開或尚未重驗的 2027 資訊寫成確定事實。

核心任務排序維持：

1. 極光機會
2. 安全
3. 體力
4. 體驗

## 第一優先：2026 團體樣本正規化

目的：把已成團歷史樣本轉成同口徑比較表，作為 2027 團體商品發布後的判斷基準。

要整理的欄位：

| 欄位 | 完成標準 |
| --- | --- |
| 團號與日期 | 保留 2026 歷史樣本來源與查核基準日 |
| 團費 | 區分標價與必要費用後總額 |
| 必要費用 | 小費、ESTA、eTA、匯率基準需分欄 |
| 極光夜數 | 同時標示 A 級與 B 級，不得混用 |
| 航班結構 | 台灣出發、跨境、內陸段與風險分段 |
| 飯店 | 標示實名飯店或「或同級」的不確定性 |
| 自由時間 | 標示市區自由時間是否足夠 |
| 恢復方案 | 延誤、取消、無法進入 YZF 時是否保護核心任務 |
| 預算判定 | 以每人 NT$150,000 作硬性比較 |

完成產物：一張「2026 歷史團體樣本正規化比較表」。

## 第二優先：2027 航班候選重查規格

目的：讓 15 組候選航班可以被重複查價，而不是停留在一次性快照。

重查時每組必須紀錄：

| 欄位 | 規則 |
| --- | --- |
| 查核日期 | 必填 |
| 來源網址 | 優先官方航空公司 |
| 日期區間 | 保留原候選日期，必要時加鄰近替代日期 |
| 路線 | TPE 到 YZF 全段列出 |
| 轉機次數 | 每方向最多 2 次為原則 |
| YVR 緩衝 | 聯票至少 3 小時；分票至少 4 小時或過夜 |
| 抵達 YZF 時間 | 晚於 20:00 不算完整極光夜 |
| 行李 | 聯票、分票、直掛風險需明列 |
| 含稅價格 | 每人經濟艙，另列行李、選位、住宿 |
| 判定 | 通過、條件通過、備援、不建議 |

完成產物：一張「2027 年 1-3 月航班候選重查表」。

## 第三優先：團體旅遊 vs 自由行同口徑總成本

目的：回答是否值得等 2027 團體商品，或先以自由行作為主方案。

比較情境：

| 情境 | 判斷方向 |
| --- | --- |
| 2026 歷史團體低價樣本 | 看團體方案在必要費用後是否仍低於 NT$150,000 |
| 2027 團體商品發布後 | 只使用公開細節，不補猜班號、飯店或成團狀態 |
| 自由行節制型雙人 | 檢查是否能保留預算緩衝 |
| 自由行舒適型雙人 | 作為目前最可能的自由行基準 |
| 自由行高舒適雙人 | 只在票價低、住宿可控時保留 |
| 單人情境 | 單房差是主要風險，需獨立判定 |

完成產物：一張「同口徑總成本與任務保護比較表」。

## 第四優先：2027 商品發布監測規則

目的：旅行社正式公布 2027 商品後，快速判斷是否進入候選，而不是重新討論標準。

任何 2027 團體商品若缺少以下資料，先標示為 `PENDING_2027_RECHECK`：

- 完整日期與天數
- 團費與必要附加費
- 航班號碼、轉機機場、轉機時間
- YZF 抵達與離開時間
- 黃刀鎮住宿夜數
- 極光安排與補看規則
- 飯店名稱或同級條件
- 取消、延誤與改目的地條款
- 行李規則
- 查核日期與來源

## 建議執行順序

1. 先產出 2026 歷史團體樣本正規化表。
2. 再產出 2027 航班重查表模板。
3. 接著補上自由行與團體同口徑成本表。
4. 最後建立 2027 商品發布後的快速篩選表。

## 決策門檻

| 判定 | 條件 |
| --- | --- |
| 可列入首選 | 低於 NT$150,000、至少 B 級極光夜，且延誤不會直接摧毀核心任務 |
| 強候選 | 低於 NT$150,000，且能接近 A 級三個完整極光夜 |
| 條件候選 | 價格可接受，但需要明確補看、延誤或 YVR 過夜方案 |
| 僅備援 | 價格、體力或證據信心不足，但仍符合基本安全 |
| 排除 | 超預算、轉機過度、核心極光夜無保護，或以未知 2027 細節作賣點 |

## 目前不應做

- 不重寫 M2 Gate。
- 不把 Backlog 項目啟用。
- 不把 2026 樣本直接視為 2027 承諾。
- 不用旅行社尚未公開的內容補猜班號、飯店或價格。
- 不把抵達夜算成完整極光夜。
