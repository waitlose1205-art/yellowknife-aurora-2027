# 台灣旅行團搜尋與比較

這個網站以通用旅行團資料模型整理台灣旅行社的公開商品，支援旅遊範圍、目的地、主題、日期、天數、價格與旅行社篩選。目前已匯入資料仍以 8 家旅行社的極光主題商品為主，網站會明確標示「部分收錄」，不宣稱涵蓋台灣所有旅行團。

## 推薦原則

只有同時符合以下條件的商品會進入綜合推薦排序：

- 來源頁身分已核對；
- 主要比較欄位完整；
- 已取得該交通方式所需的必要資訊；
- 有明確價格且在使用者預算內。

排序會考慮可報名狀態、資料完整度與相對預算節省額；極光夜數等內容保留為主題屬性，不再是所有旅行團的必要條件。來源核對只代表商品列與官方頁可對應，不代表價格、交通、日期及名額永遠有效；使用者在訂購前仍需回到來源頁確認。

## 資料流程

公開頁面只讀取 `public/data/` 內的靜態資料，不會讓訪客直接觸發旅行社網站查詢。

1. 原始旅行社資料與補充查核資料進入資料產生流程。
2. 每筆商品經過來源頁身分比對及欄位完整度分類。
3. 品質稽核輸出到 `exports/`。
4. 受保護的更新流程確認高風險筆數、宣告來源覆蓋及資料新鮮度。
5. 通過後才保留新的公開資料；失敗會回復上一版。

預期旅行社、解析方式與目前宣告收錄範圍記錄於 `scripts/pipeline/source-manifest.mjs`。旅行社存在於來源名單不等於該旅行社所有商品均已收錄；完整度必須另外檢查分頁、分類與官方結果總數。

## 本機使用

需求：Node.js `>=22.13.0`。

```bash
npm install
npm run dev
```

## 驗證

```bash
npm test
npm run lint
node scripts/audit-tour-data-readiness.mjs
```

## 更新資料

```bash
npm run data:build
npm run data:daily
npm run data:update:guarded
```

`data:build` 只會產生 `work/data-preview/` 預覽資料。`data:daily` 與 `data:update:guarded` 都會執行受保護的正式更新流程；若來源缺失、資料過期、高風險筆數超標，或可用商品明顯減少，會回復上一版公開資料。

## 發布

專案已設定 Sites，但修改不會自動發布。發布前必須完成建置、測試、資料品質稽核，並由專案負責人明確確認。
